## fix/dashboard-perf · Dashboard 性能优化

**问题来源**：导师在 [recital2.univ-nantes.fr/ptrans](https://recital2.univ-nantes.fr/ptrans/) 上发现的性能问题（2026-04-11）

### 问题描述

Dashboard 页面加载时，对每一个显示的文件都发起了完整 XML 文件的下载请求。
当前文本仓库中 XML 文件总大小约为 **19MB**，这意味着每次访问 Dashboard 页面，
浏览器都要下载约 19MB 的数据，导致页面长时间处于加载状态。

**根本原因**：Dashboard 只需要展示文件列表和提交信息（元数据），
却错误地拉取了每个文件的完整 XML 内容。

### 目标

修改 Dashboard 的数据获取逻辑：只请求文件名、提交记录等元数据，
不再在列表页下载 XML 文件的完整内容（延迟加载 / 按需加载）。

### 进度

- [ ] 定位触发 XML 下载的前端代码
- [ ] 修改为只获取元数据（文件名、最后提交信息等）
- [ ] 验证修复后网络流量显著减少
